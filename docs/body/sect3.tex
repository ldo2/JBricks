\section{Варианты заданий}

В каждом из приведенных ниже вариантов курсовой работы требуется:

\begin{itemize}
\item[1)] проанализировать полученное задание и найти способ модификации
эталонного проекта, обеспечивающий выполнение задания;
\item[2)] изобразить все изменения, сделанные в проекте, при помощи диаграмм
случаев использования и диаграммы классов;
\item[3)] реализовать новый проект, внеся необходимые изменения в интерфейсы
и реализации классов.
\end{itemize}


Все задания на включение нового типа кирпичей предполагают,
что из общего количества кирпичей двадцать процентов будут новыми.


\prb{Добавить новый вид кирпичей. Крепкий кирпич (HardBrick) исчезает с
игрового поля только при двукратном попадании в него шайбой. При первом 
попадании он должен покрыться трещинами (подмена растрового изображения).} 

\prb{ Добавить новый вид кирпичей. При попадании в такой кирпич
(PowerBrick) шайба увеличивает скорость движения на два пункта.}

\prb{ Добавить новый вид кирпичей. Кирпич (WallBrick) нельзя разбить
шайбой. Игра заканчивается, когда разбиты все кирпичи, кроме таких.}

\prb{ Добавить новый вид кирпичей. При попадании в <<липучий>> кирпич 
(StickyBrick) шайба прилипает к нему на две единицы времени, а затем
вертикально падает. Данный кирпич нельзя разбить шайбой. Игра заканчивается,
когда разбиты все кирпичи, кроме <<липучих>>.}

\prb{ Добавить новый вид кирпичей. Бронированный кирпич (ArmorBrick)
можно разбить только в том случае, когда шайба попадёт в него сверху
(со стороны потолка).}

\prb{ Добавить новый вид кирпичей. При попадании шайбы в кирпич-ловушку
(TrapBrick) снизу шайба теряется. Кирпич уничтожается при ударе
шайбы с любой другой стороны. }

\prb{ Добавить редактор уровней. Информация о положении кирпичей на игровом
поле должна содержаться в обычном текстовом файле в следующем формате:
{\tt B} означает наличие кирпича, а пробел --- отсутствие.
Файл конфигурации загружается при помощи меню в верхней панели.}

\prb{ Добавить <<мультфильмы>>, т.е. возможность воспроизведения игры
(передвижения шайбы и лопатки, разбиваниe кирпичей) после её окончания.}

\prb{ Добавить таблицу очков. В случае выигрыша программа запрашивает у
игрока имя и записывает его в файл {\tt score.txt} вместе с числом набранных
им очков, которое зависит от количества разбитых кирпичей и затраченного
времени. Таблицу очков нужно просматривать с помощью меню.}

\prb{ Добавить свойство прилипания шайбы к лопатке. Игрок нажимает клавишу
<<пробел>>, после чего при столкновении шайбы с лопаткой шайба не отскакивает,
как обычно, а остаётся на лопатке. Игрок может прицелиться, передвигая лопатку,
и затем, выбрав цель, <<выстрелить>> шайбой, повторно нажав клавишу <<пробел>>.
Этой возможностью можно воспользоваться только один раз.}

\prb{ Изменить свойства стен таким образом, чтобы при отскоке шайба вылетала
из противоположной стены, сохраняя при этом направление и скорость
(рис.~\ref{twall}).}

\prb{ Добавить <<уровни>>. Каждый раз когда игрок выигрывает, программа 
создаёт новый уровень, случайным образом расставляя кирпичи в некотором
ограниченном прямоугольнике игрового пространства.}

\prb{ Изменить реализацию классов-хранилищ {\tt PuckSupply} и {\tt BrickPile}
таким образом, чтобы они использовали стандартный интерфейс 
{\tt java.utils.Enumeration}.}

\prb{ Добавить возможность перемещения лопатки при помощи мыши.}

\prb{ Добавить <<бонусный>> кирпич (BonusBrick), при разбивании которого
появляется дополнительная шайба, которая падает вертикально вниз. 
С этого момента в игре участвуют одновременно две шайбы.}

\prb{ Добавить <<бонусный>> кирпич (BonusBrick), при разбивании которого
появляется <<бонус>> (красный круг), который затем падает вертикально вниз.
Игра заканчивается проигрышем,
если игрок не успеет поймать его при помощи лопатки. }

\prb{Реализовать правила отскока шайбы от лопатки, описанные в первом
разделе пособия (см. рис.~\ref{p3}).}

\prb{Реализовать корректный отскок шайбы при её попадании в боковые стенки
кирпича.}
